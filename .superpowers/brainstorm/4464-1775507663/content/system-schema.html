Section 3: System Schema Definition

What lives inside game_systems.schema_definition — the blueprint for each TTRPG

Schema Structure

Each game system defines: what stats exist, what resources to track, what content types are available, and how the character sheet is organized. This is admin-authored — your team curates these.

What This Enables

Different System, Same Engine

Dagger Heart would define different ability scores (e.g., Agility, Strength, Finesse), different resources (Hope, Stress), different content types (Ancestry, Class, Subclass, Domain). The expression engine and content system work identically — only the schema changes.

Creation Flow is Data

The character builder reads creation_steps to generate its wizard. Each system can define its own step order, labels, and ability score methods. One builder component serves all systems.

Extension Zones

The features sheet section is marked as an extension zone. Custom content types from DM homebrew (Ship Roles, Org Ranks) automatically appear as additional filter tabs within this section.

Curated but Flexible

Your team authors these schemas — they're not user-editable. But the content within them (SRD data, homebrew) is fully open. The schema defines the container; the community fills it.
