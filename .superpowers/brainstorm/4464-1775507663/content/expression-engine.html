Section 2: Expression Engine

How effects modify character stats — from simple bonuses to custom formulas

Three Tiers of Effects

Content (species, classes, feats, homebrew) carries effects that modify character stats. Three tiers handle simple to complex cases.

Tier 1: Static Effects
90% of homebrew needs
// +2 to Dexterity
{
  "stat": "dexterity",
  "op": "add",
  "value": 2
}
// Grant darkvision 60ft
{
  "stat": "darkvision",
  "op": "set",
  "value": 60
}

Operations: add, set, multiply, grant (proficiencies, features), max, min

Form-based UI — DM picks stat, operation, and value from dropdowns.

Tier 2: Expressions
Formulas referencing other stats
// AC = 10 + DEX mod + CON mod
{
  "stat": "armor_class",
  "op": "formula",
  "expr": "10 + mod(dexterity)
         + mod(constitution)"
}
// Spell save DC
{
  "stat": "spell_save_dc",
  "op": "formula",
  "expr": "8 + proficiency_bonus
         + mod(spellcasting_ability)"
}

Safe expression parser — arithmetic, stat references, built-in functions like mod(), max(), floor().

Text input with autocomplete. AI can help generate these.

Tier 3: Scripts
Power users — complex conditional logic
// Unarmored Defense (Barbarian)
function
compute(char) {
  if (!char.armor_equipped) {
    return {
      armor_class:
        10 + char.mod("dex")
        + char.mod("con")
    };
  }
  return {};
}

Sandboxed JS runtime (no DOM, no network, no filesystem). Character object is read-only. Returns stat modifications.

Code editor UI. AI-assisted generation from natural language.

Ship Roles Example — How It Would Work

A DM creates "Ship Role: Navigator" as homebrew content scoped to their campaign:

// content_definition for Navigator (Level 5 benefit)
{
  "content_type": "custom_feature",
  "name": "Navigator — Seasoned Voyager",
  "data": {
    "description": "Your time at the helm has sharpened...",
    "prerequisites": { "level_min": 5 }
  },
  "effects": [
    { "stat": "movement_speed", "op": "add", "value": 10 },
    { "stat": "half_cover", "op": "set", "value": "ignore" },
    { "stat": "wisdom", "op": "add", "value": 1 }
  ],
  "scope": "campaign",
  "version": 1
}

The DM fills this out via a structured form (Tier 1 effects), or writes an expression/script for complex logic. The player's character references this at version 1. If the DM nerfs the speed bonus to +5, that becomes version 2 — the player's sheet stays on v1 until they accept the update.
