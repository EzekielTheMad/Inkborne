Key Decisions

Why Supabase over alternatives?

You already know it. It gives you Postgres, auth, realtime, and storage in one service. RLS maps perfectly to content scoping. The alternatives (Prisma + separate auth + separate realtime) add integration complexity with no clear benefit for this project.

Why Zod for JSONB validation?

System schemas and content payloads are JSONB — Postgres won't validate their internal structure. Zod gives us runtime validation with TypeScript type inference. Define the shape once, get both runtime checks and compile-time types.

Expression engine lives client-side

Derived stats are computed in the browser for instant feedback. The engine is a pure TypeScript module with no DOM or network dependencies — it can also run server-side for PDF export or API responses.

Stub routes for future sub-projects

We'll create the directory structure for characters/ and campaigns/ but only build the dashboard page. This ensures routing and layout decisions are made early without building features prematurely.
